# 📋 Executing Plans — 계획 실행

작성된 구현 계획을 로드하고, 비판적으로 검토한 후, 모든 태스크를 실행합니다.

## 시작 알림
"executing-plans 워크플로우를 사용하여 이 계획을 구현합니다."라고 안내한다.

## 프로세스

### Step 1: 계획 로드 & 검토
1. 계획 파일 읽기
2. 비판적으로 검토 — 질문이나 우려 사항 식별
3. 우려 있으면: 사용자에게 먼저 제기
4. 없으면: 진행

### Step 2: 태스크 실행
각 태스크에 대해:
1. 진행 중으로 표시
2. 각 스텝을 **정확히** 따름 (계획에 상세 스텝이 있음)
3. 명시된 검증 실행
4. 완료로 표시

### Step 3: 개발 완료
모든 태스크 완료 후:
- 전체 테스트 통과 확인
- 사용자에게 선택지 제시: 병합 / PR / 유지 / 폐기

## 멈추고 도움 요청할 때
**즉시 실행을 중단:**
- 블로커 발생 (의존성 누락, 테스트 실패, 불명확한 지시)
- 계획에 치명적 빈틈
- 지시를 이해하지 못할 때
- 검증이 반복적으로 실패

**추측하지 말고 확인을 요청한다.**

## 규칙
- 계획을 먼저 비판적으로 검토
- 계획 스텝을 정확히 따름
- 검증을 스킵하지 않음
- 막혔을 때 추측하지 않고 질문
- main/master 브랜치에서 직접 구현하지 않음
